# 講師管理增強設計

> 日期：2026-03-03
> 狀態：已核准

## 目標

擴充 94Manage 講師管理功能，從目前僅有的 4 個欄位（姓名、稱謂、電話、堂薪）增加為完整的講師個資管理，包含個人資料、匯款資訊、教授科目與年級。

## 規模考量

- 目標客戶：補教協會，預估 3,000 間分校 × 10 位講師 = 30,000 筆資料
- PostgreSQL `text[]` 搭配 GIN index 在此量級效能無虞
- 採用方案 A：全部欄位加在 teachers 表，科目/年級用 `text[]` 陣列

## 資料庫變更

### 新增欄位（全部選填，不影響現有資料）

#### 個人資料
| 欄位 | 型別 | 說明 |
|------|------|------|
| `id_number` | `varchar(10)` | 身分證字號 |
| `birthday` | `date` | 生日 |
| `address` | `text` | 地址 |
| `emergency_contact` | `varchar(50)` | 緊急聯絡人姓名 |
| `emergency_phone` | `varchar(20)` | 緊急聯絡人電話 |

#### 匯款資訊
| 欄位 | 型別 | 說明 |
|------|------|------|
| `bank_name` | `varchar(50)` | 匯款銀行名稱 |
| `bank_branch` | `varchar(50)` | 分行名稱 |
| `bank_account` | `varchar(20)` | 匯款帳號 |
| `bank_account_name` | `varchar(50)` | 戶名 |

#### 教授能力
| 欄位 | 型別 | 說明 |
|------|------|------|
| `subjects` | `text[]` | 教授科目（可多選） |
| `grade_levels` | `text[]` | 教授年級（可多選） |

### 科目選項（共 16 項）

國文、英文、數學、理化、物理、化學、生物、地科、歷史、地理、公民、自然、社會、作文、閱讀、程式設計

### 年級選項（共 3 項）

國小、國中、高中

### 設計原則

- 必填欄位：僅「姓名」和「堂薪」
- 分區塊呈現，視覺清晰
- 沿用莫蘭迪色系風格
- 列表頁增加科目、年級欄位顯示

## 影響範圍

### 需修改的檔案

1. **DB Schema**: `packages/shared/src/db/schema/manage.ts` — 新增欄位定義
2. **Backend 驗證**: `apps/manage-backend/src/utils/validation.ts` — 更新 zod schema
3. **Backend 路由**: `apps/manage-backend/src/routes/w8.ts` — 更新 INSERT/UPDATE/SELECT SQL
4. **Frontend 頁面**: `apps/manage-dashboard/src/app/teachers/page.tsx` — 擴充表單和列表

### 不需修改的檔案

- 排課相關元件（不受新欄位影響）
- 薪資計算（依然基於 rate_per_class）
- 共用表 schema（不觸碰）

## API 權限控制

- 敏感欄位（身分證、匯款資訊）僅 ADMIN 和 MANAGER 角色可讀取
- 一般使用者查看講師列表時不回傳敏感欄位
